1. Cascading Style Sheets (CSS) history and evolution

CSS is a style sheet language which supports the styling of the document written by HTML or any other markup languages. CSS defines the presentation of content including colors, layout, and fonts. The firt CSS was proposed by Håkon Wium Lie in 1994. Realizing the important and strenght of styling language, he and Tim Berners-Lee introduced to public CSS1 in 1996 and soon it became a W3 recommendation for styling web page. As HTML grew, CSS became wider with variety version to meet the demand of web developers for demonstration.

  • CSS 1 The first CSS specification to become an official W3C Recommendation is CSS level 1, published on December 17, 1996. Håkon Wium Lie and Bert Bos are credited as the original developers.
  • CSS level 2 specification was developed by the W3C and published as a recommendation in May 1998. A superset of CSS 1, CSS 2 includes a number of new capabilities
  • CSS level 2 revision 1, often referred to as "CSS 2.1", fixes errors in CSS 2, removes poorly supported or not fully interoperable features and adds already implemented browser extensions to the specification.
  • Unlike CSS 2, which is a large single specification defining various features, CSS 3 is divided into several separate documents called "modules". Each module adds new capabilities or extends features defined in CSS 2, preserving backward compatibility. Work on CSS level 3 started around the time of publication of the original CSS 2 recommendation. The earliest CSS 3 drafts were published in June 1999.

2. CSS syntax

A CSS rule-set consists of a selector and a declaration block:

body {
    font-family: "Century Gothic";

    font-size: 105%;
    }

The "body" is known as the selector which is be pointed to HTML element for styling and everything inside curly brackets is know as the declaration which defines the style of selected element ("body"). "font-family" and "font-size" in this example are the properties.

"Century Gothic" and "105%" are the assigned value of the properties.

In CSS, selector can be the element name, class name or id name in HTML.

.text {
    padding:0 10px;
}

example with class name ("text").

#special {
    color:red;
}

example id name ("special").

Spacing in CSS is not matter.

p{font-size:20px;font-family:"Century Gothic";}

is the same as:

p {
    font-size: 20px;
    font-family:"Century Gothic";
}

In CSS, selectors can be grouped in order to use one specific style for many selectors.

h1, h2, p {
        text-align: center;
        color: red;
        }

In this example, all three HTML elements h1, h2 and p are the selector in CSS and they all have color red for the content and the content is aligned at the center.

3.Add CSS

When a browser reads a style sheet, it will format the HTML document according to the information in the style sheet.

There are three ways to add CSS:

Internal

The internal style is defined inside the <style> element, inside the <head> section.

<head>
    <style>
    body {
        background-color: red;
    }
    </style>
</head>

This would allow CSS to style the background-color of <body> element in HTML to have red color.

Inline

Recall from style attribute in HTML, inline styles is used by adding the style attribute to the relevant element. The style attribute can contain any CSS property. 

<p style = "color:maroon;font-size:25px" > Result: This text is style with style attribute</p>

Result: This text is style with style attribute

The style attribute above is declared to make the content of that specific <p> element has maroon color for the text and a font-size of 25px.

External

In this project, HTML page include a reference to the external CSS file so that the entire of web page can be changed in just one file.

 <link rel="stylesheet" href="../CSS/style1.css">

By using the <link> element inside the <head> element, we can add an relative link of CSS file in HTML page.

If there are external CSS and internal CSS in HTML page and at the same section, the HTML page will read the style from top to bottom and apply the style in order. Look at the below example.

<head>
    <link rel="stylesheet" type="text/css" href="../CSS/style1.css">
    <style>
    h1 {
        color: orange;
    }
    </style>
</head>
<body>
    <h1>This is an example1</h1>
</body>

Output:

This is an example1

In this example HTML reads all style applications first, and the final style is to apply orange color to the content of h1 element. Therefore, h1 content is always orange despite the position of h1 in HTML page.

If there is inline style in the above example:

<h1 style="color:red">This is an example2</h1>

Then the output:

This is an example2

The above style will not affect this <h1> element because it have the final as red color.

5.CSS Pseudo class

A CSS pseudo-class is a keyword added to a selector that specifies a special state of the selected element(s)

The pseudo class is classified by ":" placed after the selector. Pseudo class in CSS allow to style the HTML elements for example: change a button's color when the user's pointer hovers over it or change the color of visited link.

header nav ul li a:visited{
    color:#00A0C6;
}
header nav ul li a:hover {
    font-weight: bold;
    text-shadow: rgb(253, 197, 253) 2px 2px 2px;
}

The above CSS style in this project, determine the style of anchor tag <a> with pseudo class visited and hover. With the visited class, this defines visited link to have color #00A0C6 (Hexadecimal number that represent color). The hover class, as explained, creates the effect of the content whenever a pointer moves over the link. The links in the navigation bar have a shadow effect of light pink and becomes bolded.

Beside the pseudo class, there is also pseudo class which is determined by "::" after the selector. It is used to style special part of an element.

In this project, the pseudo element of <a> is used to style the bullet point of the list. 
ul.custom_list{
    list-style: none;
    
}
ul.custom_list li::before {
    content: "\2022";
    color:#00A0C6;
    font-weight: bold;
    display: inline-block; 
    width: 1em;
    margin-left: -1em;
    }

After remove the default bullet point. The before pseudo element is use to create a new bullet point placed before the content of list element.

Using the content property, we create a new bullet point with "\2022", a CSS entity which define a bullet character in web page, then the bullet is changed to be bold and placed inline with the content of the list element. The width of the boxes around the is created to be 1em so that there is a small gap between the bullet character and the list content. The left side of margin of the bullet character is assigned to be -1em so that it is move to the left side a value of 1em.

6. Image style

For a web site, the image can be an extremely important part that demonstrates the content of the web page. In CSS, images can be styled so that they are responsive and have a good layout to catch the attraction of the user.

Using the banner image as the example.

.banner img {
    width: 100%;
    height: calc(100vh - 90px);
    opacity: 0.7;
    object-fit:cover;
}

The width of the banner is assigned to be 100% so that it is responsive in differents device-width. For the height, a calc() function of CSS is used to calculate the appropriate height so that it fits a whole device screen. Since the header has a height of 90px so that in order to have a full view height of the banner, calc() function is used to calculate the view height from two different types of measure.

Opacity is a property that determines how transparent is an image (from 0 to 1.0-default).

The value of object-fit property preserves the ratio of the image and focus to cover the center of the image when it is on a small screen device. By this way, it will keep the image responsive.

7. Browser support

Each browesr supports CSS differently due to its engine, version and many other things.

In this project, the <pre> element is defined so that it can break long words and it is also designed to work on older version of browser.

pre{
    white-space: pre-wrap;      /* css-3 */
    white-space: -moz-pre-wrap;  /* Mozilla, since 1999 */
    white-space: -o-pre-wrap;    /* Opera 7 */
    word-wrap: break-word;       /* Internet Explorer 5.5+ */
}

In most case, white-space property will allow long words in <pre> tag to be break in to new line so that it fit in the example container. However, in order to make this work in other version of browsers such as Mozilla since 1999 or Internet Explorer version 5.5+.

A few technique of CSS is added in to make sure everything in <pre> tag work well in other version of browser.
